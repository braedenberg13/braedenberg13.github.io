01 — Overview

What I set out to answer

Real business decisions live and die by the questions analysts ask. For this project, I sourced a public e-commerce transactions dataset and wrote SQL to address eight questions a business analyst might actually face in a quarterly review — without touching Python or Excel.

The goal: demonstrate that SQL alone can carry a full analysis from raw data to actionable insight.

Business Question 1
Which product categories drive the most revenue, and how has that changed quarterly?
Business Question 2
What is our repeat purchase rate, and how do repeat customers compare in average order value?
Business Question 3
Which months show the strongest and weakest revenue — and what's the seasonal pattern?
Business Question 4
Are there product combinations that frequently appear together, suggesting bundling opportunities?
Methodology
Dataset acquisition & schema design

Sourced the Brazilian E-Commerce dataset (Olist) from Kaggle — 100k+ orders with items, payments, and customer tables. Loaded into PostgreSQL and validated referential integrity.

Data cleaning with SQL

Identified and handled NULL payment values, filtered test/cancelled orders, and standardized date formats using CAST and COALESCE — no external tools used.

Exploratory queries

Ran aggregate queries to understand distribution of order values, category counts, and customer frequency before writing the analytical queries.

Business-framed analysis

Each query was written to answer a specific business question, with results interpreted as a recommendation rather than just a number.

Documentation & presentation

Annotated queries with comments explaining the logic, and summarized each finding with a business takeaway.

02 — SQL Queries

Key queries, annotated

The following queries represent the core of the analysis. Each targets a specific business question and uses SQL features relevant to a professional analyst role.

query_01 — revenue_by_category_quarterly.sql Window Function 
-- Which categories drive the most revenue each quarter?
-- Uses window functions to rank within each quarter.

WITH category_quarterly AS (
  SELECT
    DATE_TRUNC('quarter', o.order_purchase_timestamp) AS quarter,
    p.product_category_name_english                         AS category,
    ROUND(SUM(oi.price + oi.freight_value), 2)              AS total_revenue,
    COUNT(DISTINCT o.order_id)                              AS order_count
  FROM orders o
  JOIN order_items oi  ON o.order_id = oi.order_id
  JOIN products p      ON oi.product_id = p.product_id
  JOIN product_category_name_translation t
                        ON p.product_category_name = t.product_category_name
  WHERE o.order_status = 'delivered'
  GROUP BY 1, 2
),
ranked AS (
  SELECT
    *,
    RANK() OVER (PARTITION BY quarter ORDER BY total_revenue DESC) AS rnk
  FROM category_quarterly
)
SELECT quarter, category, total_revenue, order_count, rnk
FROM  ranked
WHERE rnk <= 5
ORDER BY quarter, rnk;
query_02 — repeat_customer_value.sql CTE + CASE 
-- Compare one-time vs. repeat buyers: order count, avg order value.
-- Repeat = more than 1 distinct order from the same customer.

WITH customer_orders AS (
  SELECT
    c.customer_unique_id,
    COUNT(DISTINCT o.order_id)         AS total_orders,
    ROUND(AVG(p.payment_value), 2)     AS avg_order_value
  FROM customers c
  JOIN orders o      ON c.customer_id = o.customer_id
  JOIN order_payments p ON o.order_id  = p.order_id
  WHERE o.order_status = 'delivered'
  GROUP BY c.customer_unique_id
)
SELECT
  CASE WHEN total_orders > 1 THEN 'Repeat' ELSE 'One-Time' END AS customer_type,
  COUNT(*)                                                       AS customer_count,
  ROUND(AVG(avg_order_value), 2)                              AS mean_order_value,
  ROUND(AVG(total_orders), 2)                                 AS avg_orders_per_customer
FROM  customer_orders
GROUP BY 1
ORDER BY customer_count DESC;
query_03 — monthly_revenue_trend.sql Time Series + LAG 
-- Monthly revenue with MoM % change using LAG window function.
-- Surfaces seasonal peaks and slumps for planning.

WITH monthly AS (
  SELECT
    TO_CHAR(o.order_purchase_timestamp, 'YYYY-MM') AS month,
    ROUND(SUM(p.payment_value), 2)              AS revenue
  FROM orders o
  JOIN order_payments p ON o.order_id = p.order_id
  WHERE o.order_status = 'delivered'
  GROUP BY 1
)
SELECT
  month,
  revenue,
  LAG(revenue) OVER (ORDER BY month)  AS prev_month_revenue,
  ROUND(
    (revenue - LAG(revenue) OVER (ORDER BY month))
    / NULLIF(LAG(revenue) OVER (ORDER BY month), 0) * 100,
  1) AS mom_pct_change
FROM  monthly
ORDER BY month;

Business Recommendations
Launch a post-purchase loyalty incentive

With only 3.1% repeat rate but a 34% AOV premium for repeat buyers, even a small nudge (discount on second order) could materially shift revenue mix.

Invest in Health & Beauty inventory pre-Q4

This category holds the top revenue spot in every quarter, with a November spike. Pre-stocking and targeted campaigns would reduce stockouts at peak demand.

Address February revenue drop proactively

February is the lowest-revenue month every year. A targeted Valentine's Day campaign or clearance event could smooth seasonality.
